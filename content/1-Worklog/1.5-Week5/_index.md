### Week 5 Objectives:

* Implement comprehensive monitoring and observability with **Amazon CloudWatch**.
* Set up **Hybrid DNS** with Route 53 Resolver.
* Become proficient in configuring and using the **AWS CLI**.

### Tasks to be carried out this week:

| Topic | Task | Start Date | Completion Date | Reference Material |

| Amazon CloudWatch | **Monitoring & Observability** <br> - CloudWatch is a unified monitoring service that collects metrics, logs, and events across AWS, hybrid, and on-premises resources <br> - Key features: high-resolution metrics (1-second granularity), 15-month data retention, automated alarm-based actions, end-to-end observability to reduce MTTR <br> - Worked with Metrics from a sample EC2 application, viewed/created custom Metrics from Logs, configured Alarms, and built a custom Dashboard | 05/18/2026 | 05/24/2026 | AWS Study Group |

# Week 5 Achievements

## Monitoring & Observability (CloudWatch)
- Understood CloudWatch as a unified monitoring service that collects metrics, logs, and events across AWS, hybrid, and on-premises resources.
- Practiced working with Metrics, custom Metrics from Logs, configuring Alarms, and building a custom Dashboard.

## Hybrid DNS (Route 53 Resolver)
- Learned about Outbound Endpoints (forwarding queries from AWS to on-premises) and Inbound Endpoints (receiving queries from on-premises into AWS).
- Configured Resolver Rules to route DNS queries for specific domains.

## AWS CLI
- Gained a clear understanding of CLI Profiles (default and named) for managing multiple AWS environments.
- Practiced configuring the CLI via `aws configure` and working with output formats: json, yaml, yaml-stream, text, table.
